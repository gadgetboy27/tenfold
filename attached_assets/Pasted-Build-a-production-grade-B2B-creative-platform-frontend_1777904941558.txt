Build a production-grade B2B creative platform frontend called "Tenfold" — 
a modern AI content creation and social publishing tool. This is a pure 
frontend application (React + Tailwind) that connects to an existing 
backend API via environment variable VITE_API_URL.

VISUAL DESIGN BRIEF
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Aesthetic: Dark-first, cinematic, premium. Think Midjourney meets Linear.
Not corporate. Not friendly-startup-pastel. Serious creative tool energy.

Colour system:
- Background: #0A0A0A (near black)
- Surface: #111111 (cards, panels)
- Surface raised: #1A1A1A (hover states, inputs)
- Border: rgba(255,255,255,0.08) (subtle, never harsh)
- Accent primary: #7C5CFC (electric purple — brand colour)
- Accent glow: rgba(124,92,252,0.15) (for active states, focus rings)
- Text primary: #F0F0F0
- Text secondary: #888888
- Text muted: #444444
- Success: #22C55E
- Warning: #F59E0B
- Error: #EF4444

Typography:
- Display/headings: 'Syne' (Google Fonts) — geometric, distinctive
- Body/UI: 'Inter' (Google Fonts) — clean, readable
- Monospace (job IDs, credits): 'JetBrains Mono' (Google Fonts)

Motion principles:
- All state transitions: 200ms ease
- Image cards entering: staggered fade-up (60ms delay between each)
- Loading states: subtle pulse animation, never spinner
- Hover on cards: scale(1.02) + border brightens to rgba(255,255,255,0.15)
- Active/selected states: purple glow border + inner purple tint
- Panel slides: 300ms cubic-bezier(0.4, 0, 0.2, 1)

API INTEGRATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Base URL: import.meta.env.VITE_API_URL (default: http://localhost:3000)

Every API call:
- Include header: Authorization: Bearer {supabaseAccessToken}
- Include header: x-workspace-slug: {currentWorkspaceSlug}
- Handle 402: show "Insufficient credits" inline error with top-up link
- Handle 401: redirect to /login
- Handle 5xx: show toast "Something went wrong, please try again"

Key endpoints used by frontend:
GET  /api/credits/balance          → show in credit meter
POST /api/campaigns                → create campaign (returns campaignId)
GET  /api/campaigns/{id}           → poll for job + asset status
PATCH /api/campaigns/{id}/anchor   → set anchor image
POST /api/jobs                     → create video/music/script/variation job
POST /api/compositions             → run composition
POST /api/publish                  → publish to Ayrshare
GET  /api/social/profiles          → load connected platforms
GET  /api/social/connect           → get Ayrshare link URL
POST /api/credits/purchase         → open Stripe checkout

Auth: use @supabase/supabase-js for session management.
On app load: check session. If none: redirect to /login.
Store access token in memory (not localStorage).
